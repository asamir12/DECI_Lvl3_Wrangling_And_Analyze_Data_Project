# Import the json library to work with JSON data
import json
# Convert the 'tweet_id' column from the twitter_archive DataFrame to a list of strings
tweet_ids = twitter_archive['tweet_id'].astype(str).tolist()
# Create an empty list to hold selected tweet data
tweets_data = []
# Open the tweet_json.txt file (which contains one tweet JSON per line)
with open('tweet_json.txt', 'r') as file:
        # Loop through each line (i.e., each tweet in JSON format)
    for line in file:
        # Parse the JSON data in the line into a Python dictionary
        tweet = json.loads(line)
        # Check if this tweet's ID is in our list of tweet_ids
        if tweet['id_str'] in tweet_ids:
            # If it is, extract key info and append it to the list
            tweets_data.append({
                'tweet_id': tweet['id_str'],
                'retweet_count': tweet['retweet_count'],
                'favorite_count': tweet['favorite_count']
            })

Quality issues¶

  1. Missing values in the twitter_archive data frame in the next columns:
    • in_reply_to_status_id
    • in_reply_to_user_id
    • retweeted_status_user_id
    • retweeted_status_timestamp

  1. Some Tweets Lack Images: The expanded_urls column has only 2297 non-null values, out of 2356 tweets.

  2. Duplicates "retweets" In the twitter_archive DataFrame, there are 181 non-null values in retweeted_status_id, indicating retweets.

  3. Unconstant Placeholders and inaccurate Names (e.g., "a", "an", "the", "None") in the twitter_archive data frame.

  4. Wrong data type: tweet_id is stored as integer.

  5. Wrong data type: Timestamp Column in the twitter_archive data frame is an Object.

  6. The retweeted_status_id column would become useless and full of nulls after dealing with 3rd quality issue "retweets".

  7. Inaccurate data "some tweets images are not even for dogs".

Tidiness issues¶

  1. The columns:

    • doggo
    • floofer
    • pupper
    • puppo

    in the twitter_archive data frame are stored in separate columns, while they all refer to just one variable "Dog stage".

  2. All of the three DataFrames:

    • twitter_archive
    • image_preds
    • tweets_data

    Contain different aspects of the same observational unit: a single tweet. But they are stored in three separate tables, each keyed by tweet_id.

Define:¶

  • in_reply_to_status_id
